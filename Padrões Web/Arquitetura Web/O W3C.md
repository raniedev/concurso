# W3C - World Wide Web Consortium

## Processo de Documentação dos Padrões W3C

![W3C Process](imgs/w3c-process.jpg)

- **Working Draft (WD):** É o primeiro estágio de trabalho, um rascunho. O documento ainda está em desenvolvimento ativo, a  W3C publica para que a comunidade e desenvolvedores possam revisar e comentar.
- **Working Draft (Last Call):** Também chamado de **Candidate Recommendation Draft**, indica que o grupo acredita que todas as funcionalidades planejadas foram definidas, é a última chamada para revisão técnica antes de avançar para próxima etapa.
- **Candidate Recommendation (CR):** O documento já está tecnicamente completo e está sendo testado na prática, como se fosse uma fase beta do produto.
- **Proposed Recommendation (PR):** O grupo envia o documento para aprovação formal da W3C, o conteúdo está estável e pronto para se tornar um padrão oficial. Mudanças agora só acontecem por decisão administrativa (não técnica).
- **Recommendation (REC):** Estágio final, o padrão está oficialmente aprovado. É apresentado como um padrão Web reconhecido mundialmente. Navegadores, frameworks e outras ferramentas começam a implementar.

> Se um padrão for substituído ou desatualizado, ele pode ser: **Superseted**, quando substituído por um mais novo; ou **Retired**, quando retirado porque não é mais útil ou relevante.

- Checar compatibilidade de browsers com padrões web, com <a href="http://acid3.acidtests.org">Teste Acid 3</a>
- Checar compatibilidade com o <a href="https://caniuse.com">Can I Use</a>, que permite verificar vários elementos utilizados na web, como formato de imagens, elementos HTML, CSS, funcionalidades do JavaScript etc.
